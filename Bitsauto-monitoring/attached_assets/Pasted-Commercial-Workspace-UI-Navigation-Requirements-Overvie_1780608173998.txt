Commercial Workspace UI & Navigation Requirements

Overview

All newly developed commercial features should be organized into dedicated workspace modules rather than being added as individual top-level menu items or dropdown menus.

The objective is to create a clean, scalable, and business-focused user experience where each department operates from its own workspace.

⸻

Voice Trading Workspace

A dedicated top-level menu should be created:

Voice Trading

When users click on Voice Trading, they should enter a dedicated commercial workspace containing tab-based navigation.

The Voice Trading workspace should be fully commercial-focused and separate from routing, NOC, and technical operations.

Trading Authorization & Approval Workflow

Traders should only be able to offer rates within approved margin limits.

Standard Workflow

The Trading Manager or Department Head defines:

* Minimum Margin
* Allowed Discount Range
* Product Pricing Rules

The system automatically calculates:

* Cost
* Suggested Sell Rate
* Minimum Allowed Rate
* Margin %

Aggressive Pricing Requests

If a trader wants to offer a rate below the approved margin threshold:

* Submit Approval Request
* Manager receives notification
* Manager reviews request
* Manager approves or rejects

Only approved requests can be offered to customers.

Tab 2 – Rate Editor

Used for viewing and editing pricing.

Features:

* Search Destination
* Search Product
* Search Customer

Rate Editor Limitations

The Rate Editor should support:

* Single Destination Rate
* Single Product Rate
* Breakout Rate

It should not be used for mass customer distributions.

UI Design Standard

All future commercial modules should follow the same structure.

Instead of creating multiple top-level menus:

Recommended Structure

Voice Trading
├── Dashboard
├── Deals
├── Route Board
├── Notifications
├── Reports

Rate Management
├── Dashboard
├── Rate Editor
├── Send Rates
├── Notifications
├── Vendor Impact Analysis
├── 7-Day Notice Queue
├── Product History
├── Approvals
├── Reports

Customer Management
├── Dashboard
├── Customers
├── Pricing
├── Notifications
├── Reports

This approach keeps the platform organized, scalable, and easy to use while ensuring that all commercial functions remain grouped within dedicated workspaces rather than being scattered across multiple dropdown menus.
