1. Incident Lifecycle State Machine (FOUNDATION)
File: server/incidents/stateMachine.ts
Core states
export type IncidentState =
  | "active"
  | "acknowledged"
  | "investigating"
  | "mitigating"
  | "resolved";
Allowed transitions (IMPORTANT — audit-safe)
active → acknowledged
active → investigating

acknowledged → investigating
acknowledged → mitigating

investigating → mitigating
investigating → resolved

mitigating → resolved
Hard rules (critical)
1. No direct jump to resolved from active

Prevents silent auto-resolution bugs.

2. Resolved is terminal

No edits allowed except:

reopen → only if explicitly triggered
3. Reopen rule (safe)
resolved → active (reopen event required)
Event emission (IMPORTANT for Phase 2 timeline)

Every transition emits:

{
  incidentId,
  fromState,
  toState,
  timestamp,
  actor
}

👉 This becomes the backbone of the timeline builder.
