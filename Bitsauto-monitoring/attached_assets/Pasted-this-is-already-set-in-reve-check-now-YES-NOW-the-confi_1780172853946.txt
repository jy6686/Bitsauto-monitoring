You identified a VERY important architectural limitation of REVE HTTP Query mode.

Current Mode

You are using:

DLR = Query
Method = GET

In this mode:
REVE behaves like:

GET https://your-url?param=value

This mode:

does NOT support auth headers
does NOT support bearer token
does NOT support API secret section
does NOT support JSON body auth

That is why:

API key fields disappear.

This is expected behavior.

What REVE Is Doing Right Now

Effectively:

GET /api/bhaoo/receive?to=923xxxx&smsText=1234

No:

Authorization header
API secret
bearer token
Is This BAD?

For testing:

NO

For production:

YES, risky.

Because anyone who discovers endpoint could:

spam Voice OTP
trigger fake calls
abuse routing
