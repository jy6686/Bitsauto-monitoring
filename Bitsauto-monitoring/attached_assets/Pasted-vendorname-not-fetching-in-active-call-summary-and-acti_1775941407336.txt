Here's exactly what was fixed and how it works now:

Root cause: Sippy's XML-RPC listActiveCalls API returns I_CONNECTION (a numeric ID like "2") but never returns a vendor name. The old code tried to read vendor_name from the response — which doesn't exist in XML-RPC mode — so it was always undefined.

The fix:

Startup (5s delay)
  → fetch all vendors (Callntalk iVendor=2, TALK iVendor=3)
  → for each vendor fetch its connections
  → build cache: { "2" → "Callntalk", "3" → "TALK", ... }
  → refresh every 30 minutes automatically
Live call with I_CONNECTION = "2"
  → vendor = c.vendor (portal mode, already set) 
          OR connectionVendorCache.get("2")  ← NEW fallback
          = "Callntalk"  ✓

Both the Active Call Summary vendor filter and the Active Call Details vendor field will now show the correct vendor name. The cache also auto-refreshes every 30 minutes, so if you ever add new vendor connections they'll show up without a restart.
