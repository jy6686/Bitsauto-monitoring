🔌 🧠 SIPPY API HEALTH CHECKER (MODULE DESIGN)
🎯 PURPOSE

Continuously monitor Sippy Soft API health to ensure:

API is reachable
Authentication is valid
Responses are correct
Latency is acceptable
No silent failures

This will support your Fix Button system + Live monitoring dashboard

⚙️ 1. CORE HEALTH CHECK ENDPOINT
Create API:
GET /api/sippy/health
🧾 Response Example
{
  "status": "healthy",
  "auth": "valid",
  "latency_ms": 320,
  "last_success_call": "2026-04-19T18:20:00Z",
  "error": null
}
🧠 2. HEALTH CHECK LOGIC (BACKEND)
Step-by-step flow:
✅ Step 1 — Test Authentication
Use API USER/PASS
Attempt lightweight Sippy API call (safe method like balance or ping)
await sippyClient.login(API_USER, API_PASS);
✅ Step 2 — Test API Response

Call lightweight endpoint:

balance check
system status
CDR sample fetch (limit 1)
✅ Step 3 — Measure Latency
const start = Date.now();
await sippyClient.ping();
const latency = Date.now() - start;
✅ Step 4 — Validate Data Integrity

Check:

response exists
no null/empty payload
correct format (XML/JSON parsed correctly)
✅ Step 5 — Error Handling

Detect:

Issue	Meaning
401	Auth failed
timeout	network issue
empty response	API broken
invalid XML	parsing issue
⚠️ 3. HEALTH STATUS RULES
if (authFailed) status = "unhealthy";
else if (timeout) status = "degraded";
else if (latency > 2000) status = "slow";
else status = "healthy";
