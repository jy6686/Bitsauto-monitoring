The correct deployment sequence

You should treat this as a staged database migration.

Step 1

Run the duplicate audit on production.

You already know the query:

SELECT
    dial_prefix,
    COUNT(*)
FROM global_destinations
WHERE dial_prefix IS NOT NULL
  AND dial_prefix <> ''
GROUP BY dial_prefix
HAVING COUNT(*) > 1;

Confirm production has the same duplicate pattern.

Run the same archive + cleanup migration you successfully tested on development.

That includes:
