About Range-Based Prefix Expansion

I would not implement it yet.

This is why.

Your screenshot shows values like:

12645391264540

Before deciding how to expand them, answer this question:

How do vendors encode multiple prefixes?

Different vendors use:

1264;12645;12646

Others use:

1264,12645

Others use:

1264-1269

Others use:

12645391264540

That last example might be:

concatenated prefixes,
malformed data,
or a vendor-specific notation.

I would first build a Prefix Parsing Engine that supports multiple vendor formats instead of hardcoding Telstra's format.
