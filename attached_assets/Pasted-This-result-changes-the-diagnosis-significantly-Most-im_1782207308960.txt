Bigger architectural issue

Your system currently has:

Product Assignment = Database
Account Catalog    = Memory Cache

Those should never be separate authorities.

What I would build

Instead of:

customer_product_assignments
      ∩
__bitsautoAccountCache

create a persistent account catalog table.

Example:

CREATE TABLE account_registry (
    i_account      INTEGER PRIMARY KEY,
    username       TEXT,
    tariff_id      INTEGER,
    status         TEXT,
    last_seen_at   TIMESTAMP,
    source         TEXT
);

Populate it from:

Admin API listAccounts()

during sync.

Then every module uses:

account_registry

instead of memory cache.
