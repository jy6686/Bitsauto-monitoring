The Termination Report should resolve identities in this exact order:

Step 1 — Direct iConnection match (preferred)

If CDR contains:

cdr.iConnection

then:

connection = connectionCache[iConnection]
vendor = vendorCache[connection.vendorId]

Output:

VendorName / ConnectionName
Step 2 — Connection name match from cached live routes

If iConnection missing:

Use existing routing/connection caches already loaded in:

Connections page
Vendors page
Routing Manager

Possible deterministic joins:

cdr.remoteIp
cdr.terminationHost
cdr.routeName
cdr.vendor
SIP gateway name

against:

cachedConnections[]

NOT country matching.

Step 3 — Vendor-only fallback

If connection unresolved but vendor field exists:

Show:

VendorName / Unknown Connection

instead of plain Unknown.

Step 4 — Honest unknown

Only after all deterministic joins fail:

Unknown Vendor / Unknown Connection
