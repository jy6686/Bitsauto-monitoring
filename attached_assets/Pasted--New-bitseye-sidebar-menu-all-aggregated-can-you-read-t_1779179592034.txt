As soon as a call appears in Sippy active calls:

call enters liveCallsCache

Example:

{
  "call_id": "abc123",
  "client": "Centredtel",
  "vendor": "BICS",
  "country": "Bangladesh",

}
Step 3 — Real-Time Bucket Engine

Immediately aggregate:

Country → Bangladesh → +1 concurrent
Client → Centredtel → +1 concurrent
Vendor → BICS → +1 concurrent

Now ALL graphs update automatically.

Old Thinking
Graphs read from reports/CDRs
Correct Thinking
Graphs are generated from active call state
What Happens When One Call Arrives
Example

New call:

This is exactly how telecom NOC systems behave.

What Happens When Call Ends

The call is removed from:

liveCallsCache

Then:

all counters decrement automatically

Graphs instantly reflect the new concurrent level.

It is not querying history.

It is continuously rebuilding:

concurrent state snapshots
