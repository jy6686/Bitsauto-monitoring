🧩 1. System Architecture (simple + scalable)
BitsEye now has 2 engines:
🔷 A. Graph Engine (Relationship Layer)

Builds a live node-edge map from existing data:

Nodes

KAM
Client (Account)
Vendor (Carrier)
Destination
Breakout (Traffic pattern)
Route Group (optional future)
Incident (temporal node)

Edges

traffic_volume
ASR / ACD / PDD impact
revenue flow
routing dependency
incident correlation
ledger action linkage (FREEZE, ROUTE_CHANGE, etc.)
🔶 B. Investigation Engine (Context Layer)

When user clicks any node:

It overlays:

ledger timeline (intent-based grouping)
incidents in ± time window
carrier score delta
routing changes
CDR anomaly spikes

👉 This is your “why did this happen?” layer

You do NOT need new data sources.

Just unify existing ones:

Source systems already used:
action_ledger (C2 + routing)

carrier_scores
cdr_cache

Node = entity (client/vendor/destination/kam)

  - incident correlation
