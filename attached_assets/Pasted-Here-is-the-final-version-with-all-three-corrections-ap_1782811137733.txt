Step 2: Identify clusters by characteristics (not ID ranges)
Define clusters as follows:

Cluster A (old generation): rows where dial_prefix LIKE '+%' — the +prefix import format
Cluster B (new generation): rows where dial_prefix NOT LIKE '+%' — the bare prefix import format

A duplicate pair exists when regexp_replace(cluster_a.dial_prefix, '^\+', '') = cluster_b.dial_prefix.
Classify each pair:

exact_duplicate — same normalized prefix AND names are substantially similar (normalized text match >80%)
parent_child — Cluster A normalized prefix is shorter than Cluster B (e.g. 92 vs 9230) — Cluster A is a broader routing prefix, not the same destination
name_only — normalized prefix matches but names differ significantly
manual_review — cannot be confidently classified by the above rules

Set recommended_action:

MERGE — exact_duplicate confirmed, safe to remap Cluster A FKs to Cluster B then delete Cluster A
KEEP_BOTH — parent_child, genuinely different destinations at different prefix depths
MANUAL_REVIEW — insufficient confidence
