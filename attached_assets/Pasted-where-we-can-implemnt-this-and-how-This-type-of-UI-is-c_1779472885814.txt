This type of UI is commonly called a Grid Menu or Card-Based Grid Layout.

More specifically in mobile UI/UX design, it can also be described as:

•⁠  ⁠Category Grid Menu
•⁠  ⁠Tile Menu
•⁠  ⁠Dashboard Grid
•⁠  ⁠Card Navigation Layout

The screenshot from the Apple App Store uses large interactive cards/tiles arranged in a 2-column grid for category navigation.

Key characteristics:

•⁠  ⁠Large tappable cards
•⁠  ⁠Visual icons/illustrations
•⁠  ⁠Category-based navigation
•⁠  ⁠Responsive grid arrangement
•⁠  ⁠Card-style UI design

This pattern is very common in:

•⁠  ⁠App stores
•⁠  ⁠Streaming apps
•⁠  ⁠Admin dashboards
•⁠  ⁠Smart TV interfaces
•⁠  ⁠E-commerce categories

If you are discussing it with a designer or developer, you can simply call it:

“A card-based category grid menu”
or
“A tile/grid navigation UI”

This UI pattern is PERFECT for your platform — but NOT for the entire platform.

The key is:

where to use it

because card-grid navigation works best for:

workspace entry
workflow hubs
mobile-friendly landing
operational launchpads

NOT:

dense engineering workflows
always-visible operational tools
deep nested telecom controls
